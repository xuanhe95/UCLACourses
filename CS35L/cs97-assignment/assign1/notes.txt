I find two possible answers, the key is that linuxwords and sortedwords both contain character that are not part of "A-Za-z", so for the original command, after tr -cs 'A-Za-z', only alphabets are left, but for sortedwords those special characters such as '-' '!' '/' are still there.
So there are two approaches: modify the first tr -cs to make sure the input file still contain those special character, or modify the sortedwords to eliminate those special character
tr -cs "A-Za-z0-9-',\!/" '[\n*]'| sort -u | comm -23 - sorted.words
## this retains all the special character in the inputfile such as ',', '!', '-'
or
tr -cs 'A-Za-z' '[\n*]' | sort -u | comm -23 - <(sort -u <(tr -cs 'A-Za-z' '[\n*]' <sorted.words))
## this modify the sorted.words to only contain alphabet
